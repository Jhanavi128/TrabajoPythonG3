* Objetivos
- Aprender a bootear un sistema operativo Linux desde un medio
  externo como USB, comprendiendo el proceso de creación de un
  dispositivo booteable y su ejecución en modo /Live/.

- Comprender la secuencia de arranque de un computador, identificando
  las diferencias y el rol que cumplen la BIOS y la UEFI dentro de
  dicho proceso.

- Identificar y utilizar comandos de Linux (~lscpu~, ~lsb_release~ y
  ~lspci~) para obtener información del hardware y del sistema
  operativo desde la terminal.

* Instrucciones
1. Siga el tutorial para la creación de un booteable de Ubuntu como se
   indica en la unidad 9 en el archivo Arranque Ubuntu[fn:2].

2. Realice un respaldo del código de /BitLocker/ utilizando un command
   prompt de Windows con permisos de administrador[fn:1]:

   #+begin_src shell
    manage-bde-protectors C:-get
   #+end_src
   
3. Utilizando su computador o el del laboratorio, siga los pasos
   indicados para obtener el booteo desde el USB con el Sistema
   Operativo de Ubuntu

4. Realice el boot de Ubuntu desde la memoria USB externa y
   seleccione la opción de /Probar Sistema Operativo/.
5. Una vez ingresado active un terminal de Ubuntu: ~Ctrl+ALT+T~ y
   obtenga las características del computador mediante la ejecución de
   los comandos: ~lscpu~, ~lsb_release -a~ y ~lspci~. Obtenga capturas
   de la ejecución de los comandos. Luego abra este archivo en WSL y
   ejecute los comandos desde los bloques shell. Para insertar un
   bloque puede usar la combinación ~C-c C-,~, escoja la opción ~src~
   e indique que es del tipo shell.

* Consulta
Describa que hacen y para qué sirven los comandos utilizados en la práctica

- lscpu :: Es un comando que muestra información sobre la arquitectura de la
  CPU obtenida directamente desde el sistema de archivos ~/proc/cpuinfo~
  y desde llamadas al sistema, entre los datos que reporta
  se encuentran: la arquitectura (x86_64, ARM, etc.), el modo de
  operación (32 o 64 bits), el número de núcleos (/cores/), hilos por
  núcleo, sockets, el fabricante y modelo del procesador, la
  frecuencia de reloj, y la jerarquía de memoria caché (L1, L2, L3).
  Es útil para diagnosticar el hardware disponible, verificar
  compatibilidad de software con la arquitectura del procesador y
  para tareas de optimización o virtualización.
- lsb\_release -a :: Es un comando que despliega información sobre la distribución de
  Linux instalada, basada en el estándar /LSB/ que es Linux Standard Base,
  el cual busca unificar la estructura interna entre distintas
  distribuciones, la opción ~-a~ muestra todos los siguientes datos
  disponibles: el /Distributor ID/, la descripción completa, el número de versión (/Release/) y
  el nombre en clave (/Codename/) de dicha versión, es útil para
  identificar rápidamente qué distribución y versión se está
  utilizando, algo esencial al momento de instalar paquetes,
  reportar errores o verificar compatibilidad de software.
- lspci :: Es un comando que lista todos los dispositivos conectados a los
  buses PCI y PCIe del sistema, tales como la tarjeta gráfica, controladores
  de red, controladores USB, tarjetas de sonido y otros periféricos integrados en la
  placa base, este comando obtiene esta información consultando directamente el
  árbol de dispositivos del kernel en ~/sys/bus/pci~, es bastante útil para
  identificar el hardware instalado, diagnosticar problemas de
  drivers o controladores faltantes y verificar la compatibilidad
  de dispositivos con el sistema operativo.
